## Overview

Refactron provides a comprehensive CLI for code analysis, refactoring, AI features, and more.

## Core Commands

### analyze

Analyze code for issues.

```bash
refactron analyze <path> [options]
```

**Options:**
- `--detailed` - Show detailed analysis output
- `--summary` - Show summary only
- `--config FILE` - Use custom config file

**Examples:**
```bash
# Analyze single file
refactron analyze myfile.py

# Analyze directory with details
refactron analyze myproject/ --detailed
```

---

### refactor

Preview or apply refactoring suggestions.

```bash
refactron refactor <path> [options]
```

**Options:**
- `--preview` - Preview changes without applying
- `--type TYPE` - Filter by operation type (can use multiple)
- `--risk-level LEVEL` - Filter by risk: safe, low, moderate, high
- `--ai` - Enable AI-powered refactoring
- `--feedback` - Enable interactive feedback

**Examples:**
```bash
# Preview all refactorings
refactron refactor myfile.py --preview

# Apply specific types only
refactron refactor myfile.py --type extract_constant --type add_docstring

# AI-powered refactoring
refactron refactor myfile.py --ai --preview
```

---

### autofix

Apply automated fixes.

```bash
refactron autofix <path> [options]
```

**Options:**
- `--preview` - Preview auto-fixes
- `--apply` - Apply auto-fixes
- `--no-backup` - Don't create backups

**Examples:**
```bash
# Preview auto-fixes
refactron autofix myfile.py --preview

# Apply safe fixes
refactron autofix myfile.py --apply
```

---

### report

Generate analysis reports.

```bash
refactron report <path> [options]
```

**Options:**
- `--format FORMAT` - Output format: json, html, text
- `-o FILE` - Output file path

**Examples:**
```bash
# JSON report
refactron report myproject/ --format json -o report.json

# HTML report
refactron report my project/ --format html -o report.html
```

## Authentication

### auth

Manage authentication state and telemetry.

```bash
refactron auth <command> [options]
```

**Commands:**
- `login` - Interactive device-code login
- `logout` - Remove local credentials
- `status` - Show current auth status and plan
- `telemetry` - Manage usage data collection (`--enable`, `--disable`, `--status`)

**Examples:**
```bash
# Check status
refactron auth status

# Login (top-level alias also works: refactron login)
refactron auth login

# Disable telemetry
refactron auth telemetry --disable
```

## Workspace & Repository

### repo

Manage connections to GitHub repositories.

```bash
refactron repo <command> [options]
```

**Commands:**
- `list` - List repositories connected via WebApp
- `connect` - Link a repository to a local path (auto-clones if no path provided)
- `disconnect` - Remove local mapping and optionally delete files

**Options (connect):**
- `--path PATH` - Local directory to link (default: `~/.refactron/workspaces/`)
- `--api-base-url URL` - Custom API endpoint

**Examples:**
```bash
# List available repos
refactron repo list

# Clone and connect a new repo
refactron repo connect user/my-repo

# Link current directory to a repo
refactron repo connect my-repo --path .
```

## Analysis & AI

### analyze

Analyze code for patterns, smells, and complexity.

```bash
refactron analyze <path> [options]
```

**Options:**
- `--detailed` - Show full issue details
- `--summary` - Show summary statistics only
- `--config FILE` - Path to `.refactron.yaml`

---

### rag

Manage the Retrieval-Augmented Generation (RAG) index for AI context.

```bash
refactron rag <command> [options]
```

**Commands:**
- `index` - Index the current workspace
- `search` - Semantic search across the codebase
- `status` - Check indexing progress

**Options (index):**
- `--summarize` - Generate AI summaries for each file (uses LLM)
- `--force` - Re-index all files even if unchanged
- `--background` - Run indexing in a detached process

---

### suggest

Get AI-powered improvement suggestions for specific code blocks.

```bash
refactron suggest <path> --line <n> [options]
refactron repo <command> [options]
```

**Commands:**
- `list` - List connected repositories
- `connect` - Connect a repository (auto-detects local git repo, or clones from GitHub)
- `disconnect` - Disconnect a repository

**Options:**
- `--ssh` - Clone using SSH instead of HTTPS (for connect API fallback)
- `--delete-files` - Delete local files (for disconnect)

**Examples:**
```bash
# List repos
refactron repo list

# Connect current local git repository (offline)
refactron repo connect

# Clone and connect remote repository
refactron repo connect my-repo

# Clone via SSH instead of HTTPS
refactron repo connect --ssh my-repo

# Disconnect repo
refactron repo disconnect my-repo
```

## Refactoring

### refactor

Apply or preview automated refactoring.

```bash
refactron refactor <path> [options]
```

**Options:**
- `--ai` - Enable LLM-powered refactoring logic
- `--type <type>` - Filter by: `extract_method`, `magic_number`, `simplify_conditionals`, `reduce_parameters`, `add_docstring`
- `--risk-level <level>` - Filter by risk: `safe`, `low`, `moderate`, `high`

---

### autofix

Apply common fixes automatically (non-AI based).

```bash
refactron autofix <path> [options]
```

## Patterns & Feedback

### feedback

Provide feedback on AI suggestions to improve future results.

```bash
refactron feedback <operation-id> --action <action> [options]
```

**Actions:** `accepted`, `rejected`, `ignored`

---

### patterns

Manage the local pattern-learning engine.

```bash
refactron patterns <command> [options]
```

**Commands:**
- `analyze` - Show project-specific acceptance rates
- `recommend` - Generate tuning suggestions
- `tune` - Apply weights and enable/disable rules based on feedback
- `profile` - View current project tuning profile

---

## Infrastructure

### generate-cicd

Generate integration templates for various platforms.

```bash
refactron generate-cicd <type> [options]
```

**Types:** `github`, `gitlab`, `pre-commit`, `all`

**Options:**
- `--python-versions` - Comma-separated list (e.g., `3.10,3.11,3.12`)
- `--max-errors N` - Fail build if errors exceed N

---

### init

Initialize a new Refactron project with a configuration template.

```bash
refactron init [options]
```

**Options:**
- `--template <name>` - Templates: `base`, `django`, `fastapi`, `flask`

---

## Global Options

Available for all commands:

- `--help` - Show help message
- `--version` - Show version
- `--config FILE` - Use custom config file
- `--log-level LEVEL` - Set log level (DEBUG, INFO, WARNING, ERROR)

**Examples:**
```bash
# Show version
refactron --version

# Custom config
refactron analyze myfile.py --config custom-config.yaml

# Debug logging
refactron analyze myfile.py --log-level DEBUG
```

## Environment Variables

```bash
# Disable color output
export REFACTRON_NO_COLOR=1

# Set log level
export REFACTRON_LOG_LEVEL=DEBUG

# Custom config path
export REFACTRON_CONFIG=/path/to/.refactron.yaml

# API keys
export GROQ_API_KEY=your-key
export REFACTRON_API_KEY=your-key
```

