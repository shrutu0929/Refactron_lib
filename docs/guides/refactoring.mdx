## Overview

Refactron provides intelligent refactoring suggestions with safety previews, risk scoring, and automated fixes. All refactorings include backup and rollback capabilities.

## Basic Refactoring

### Preview Changes

Always preview refactoring suggestions before applying:

```bash
refactron refactor myfile.py --preview
```

### Apply Refactoring

Apply all suggested refactorings:

```bash
refactron refactor myfile.py
```

<Warning>
  Refactoring modifies your code. Always preview changes first and ensure you have backups or version control.
</Warning>

## Available Refactorers

<AccordionGroup>
  <Accordion title="Extract Constant">
    Replaces magic numbers with named constants

    ```python
    # Before
    def calculate_tax(amount):
        return amount * 0.18

    # After
    TAX_RATE = 0.18

    def calculate_tax(amount):
        return amount * TAX_RATE
    ```
  </Accordion>

  <Accordion title="Add Docstring">
    Adds missing docstrings to functions and classes

    ```python
    # Before
    def calculate_total(items):
        return sum(item.price for item in items)

    # After
    def calculate_total(items):
        """Calculate total price for list of items.

        Args:
            items: List of items with price attribute

        Returns:
            Total sum of item prices
        """
        return sum(item.price for item in items)
    ```
  </Accordion>

  <Accordion title="Simplify Conditionals">
    Refactors complex conditional expressions

    ```python
    # Before
    if not (x < 10 or x > 20):
        process()

    # After
    if 10 <= x <= 20:
        process()
    ```
  </Accordion>

  <Accordion title="Reduce Parameters">
    Reduces function parameters using dataclasses or dicts

    ```python
    # Before
    def create_user(name, email, age, city, country):
        pass

    # After
    from dataclasses import dataclass

    @dataclass
    class UserData:
        name: str
        email: str
        age: int
        city: str
        country: str

    def create_user(user: UserData):
        pass
    ```
  </Accordion>
</AccordionGroup>

## Filter by Risk Level

```bash
# Only apply safe refactorings
refactron refactor myfile.py --risk-level safe

# Apply safe and low-risk refactorings
refactron refactor myfile.py --risk-level low
```

## Specific Refactorings

Apply only specific types of refactoring:

```bash
# Only extract constants
refactron refactor myfile.py --type extract_constant

# Multiple types
refactron refactor myfile.py --type extract_constant --type add_docstring
```

## Python API

```python
from refactron import Refactron

refactron = Refactron()

# Preview refactoring
result = refactron.refactor("myfile.py", preview=True)
result.show_diff()

# Apply specific refactorings
result = refactron.refactor(
    "myfile.py",
    preview=False,
    operation_types=["extract_constant", "add_docstring"]
)

if result.success:
    print(f"Applied {len(result.operations)} refactorings")
    print(f"Backup: {result.backup_path}")
```

## Auto-Fix

The `autofix` command applies safe, automated fixes:

```bash
# Preview auto-fixes
refactron autofix myfile.py --preview

# Apply auto-fixes
refactron autofix myfile.py --apply
```

Auto-fix includes 14 automated fixers for common issues.

## Backup and Rollback

Every refactoring creates automatic backups:

### List Rollback Sessions

```bash
refactron rollback --list
```

### Rollback Specific Session

```bash
refactron rollback --session <session-id>
```

### Python API Rollback

```python
from refactron.autofix.file_ops import FileOperations

file_ops = FileOperations()

# Rollback one file
file_ops.rollback_file("myfile.py")

# Rollback all changes
file_ops.rollback_all()
```

## Refactoring Workflow

<Steps>
  <Step title="Analyze">
    First, analyze your code to identify issues
    ```bash
    refactron analyze myproject/
    ```
  </Step>

  <Step title="Preview">
    Preview refactoring suggestions
    ```bash
    refactron refactor myproject/ --preview
    ```
  </Step>

  <Step title="Review">
    Review the diff output and risk scores
  </Step>

  <Step title="Apply">
    Apply refactorings you want to keep
    ```bash
    refactron refactor myproject/ --type extract_constant
    ```
  </Step>

  <Step title="Test">
    Run your tests to ensure nothing broke
  </Step>

  <Step title="Rollback (if needed)">
    Rollback if something went wrong
    ```bash
    refactron rollback --session <id>
    ```
  </Step>
</Steps>

## Configuration

Configure refactoring behavior in `.refactron.yaml`:

```yaml .refactron.yaml
# Enable specific refactorers
enabled_refactorers:
  - extract_constant
  - add_docstring
  - simplify_conditionals
  - reduce_parameters

# Risk tolerance
max_risk_score: 0.5  # Don't apply refactorings with risk > 0.5

# Backup settings
create_backups: true
backup_dir: .refactron/backups
```

## Real-World Tutorial: Modernizing a Legacy Module

In this tutorial, we'll walk through a common scenario: modernizing a legacy utility module that lacks documentation, uses magic numbers, and has complex logic.

### 1. The Starting Point (`legacy_utils.py`)

```python
def calc(a, b, c):
    if not (a < 0 or a > 100):
        res = a * 0.05
    else:
        res = a * 0.1
    return res + (b * 3.14)
```

### 2. Step 1: Automated Analysis

Run an initial analysis to identify specific smells and complexity issues:

```bash
refactron analyze legacy_utils.py --detailed
```

Output highlights:
- `MagicNumber`: `0.05`, `0.1`, `3.14`
- `Complexity`: 2 (Low, but can be simplified)
- `MissingDocstring`: Function `calc` lacks documentation

### 3. Step 2: Safe Refactoring (Preview)

We'll start by applying safe refactorings like constant extraction and docstring generation.

```bash
refactron refactor legacy_utils.py --type extract_constant --type add_docstring --preview
```

Refactron suggests:
- Extracting `0.05` to `DISCOUNT_RATE_A`
- Extracting `3.14` to `PI`
- Adding a descriptive Google-style docstring.

### 4. Step 3: Logic Simplification

Next, let's simplify the conditional logic:

```bash
refactron refactor legacy_utils.py --type simplify_conditionals --apply
```

### 5. Final Result

The module is now readable, documented, and maintainable:

```python
"""Legacy utility module refactored for readability."""

DISCOUNT_RATE_LOW = 0.05
DISCOUNT_RATE_HIGH = 0.1
PI = 3.14

def calc(value, modifier, multiplier):
    """
    Calculate adjusted value based on range and modifiers.
    
    Args:
        value: The base amount to calculate from.
        modifier: Secondary adjustment factor.
        multiplier: Circular adjustment factor.
    """
    if 0 <= value <= 100:
        res = value * DISCOUNT_RATE_LOW
    else:
        res = value * DISCOUNT_RATE_HIGH
        
    return res + (modifier * PI)
```

## Advanced Recipes

### Recipe: Project-Wide Docstring Standardization

Use this script to ensure every function in your project has a docstring before a release.

```python
from refactron import Refactron
from pathlib import Path

refactron = Refactron()

# Target only the source directory
for py_file in Path("src/").rglob("*.py"):
    refactron.refactor(
        str(py_file),
        operation_types=["add_docstring"],
        risk_level="safe",
        preview=False
    )
```

### Recipe: Enforcing Low Complexity in CI

Integrate this into your test suite or CI script to block PRs that exceed complexity limits.

```python
import sys
from refactron import Refactron
from refactron.core.config import RefactronConfig

# Strict configuration for CI
config = RefactronConfig(max_function_complexity=8)
refactron = Refactron(config)

analysis = refactron.analyze("src/")
complexity_issues = [i for i in analysis.issues if i.category == "COMPLEXITY"]

if complexity_issues:
    print(f"❌ Failure: {len(complexity_issues)} functions exceed complexity limits!")
    for issue in complexity_issues:
        print(f"  - {issue.file_path}:{issue.line_number} -> {issue.message}")
    sys.exit(1)

print("✅ Code quality gate passed!")
```
